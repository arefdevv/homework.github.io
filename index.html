Introduction to OSI Model

OSI Model stands for Open Systems Intercommunication, is a theoretical model of how network communication works. It consists of 7 layers
and each layer represents a specific function. The model allows different devices to communicate with one another while following the rules.

1. Physical Layer

Physical Layer deals with hardware transmission
of data over a physical medium such as ethernet
cable. The main focus of this layer is the physical
connection between devices. A real life example
of this at FraudWatch would be ethernet cable that
connects to a laptop, which facilitates transmission
of data between devices.

2. Data Link Layer

Data Link layer is responsible for the
data to be transmitted reliably between
two connected nodes. One example of
this at FraudWatch would be a FraudWatch
laptop's MAC Address which data link layer
uses to determine the source and destination of
each frame.

3. Network Layer

Network layer is responsible for routing and
forwarding data packets between different
networks. The main function of Network layer
is to establish, maintain and terminate connections
ensuring data is delivered to its correct destination.
An example of this would be the routers at FraudWatch office?

4. Transport Layer

Transport layer's primary function is to provide
end to end communication and data transfer
between devices. Transport layer ensures that
data is transferred accurately, in order and
reliably. One example of this at Fraudwatch
would be a SOC team member accessing PP
from their laptop, the transport layer ensures
that the communication between the user's device
and the server is secure and reliable.

5. Session Layer

The main function of session layer is
to establish, terminate and manage sessions
on application running on different devices.
Session layer enables applications to communicate
and synchronize their activities ensure a
seamless data exchange.

6. Presentation Layer

Presentation layer is responsible for preparing
data for the application layer. It essentially
acts as a translator that is translating data
between application layer and lower level
layers (session and ect.) so that both the
receiver and sender can understand.

7. Application Layer

The application layer is used by end-user
software such as web browsers and email clients.
It provides protocols that allow software to send
and receive information and present meaningful data
to users.
